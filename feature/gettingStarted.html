Next.js

Next.js es un framework de React que facilita la creación de aplicaciones web completas. Al desplegarse en Vercel, ofrece ventajas como configuración cero, escalabilidad automática, mayor rendimiento y disponibilidad global.

Entre sus principales características están:

  • ISR (Incremental Static Regeneration): actualiza contenido sin necesidad de redeploy.
  • SSR (Server-Side Rendering): renderiza contenido dinámico en cada petición.
  • Streaming: carga datos por partes para mejorar la velocidad.
  • Optimización automática: de imágenes, fuentes y tarjetas sociales.
  • Herramientas extra: Middleware, Draft Mode, Web Analytics y métricas de rendimiento.

En resumen, usar Next.js en Vercel significa tener despliegues rápidos, optimización integrada y un mejor rendimiento global para aplicaciones modernas.

Vercel: Conceptos Fundamentales de Proyectos y Despliegues

Un proyecto en Vercel representa la aplicación que usted ha desplegado. Múltiples proyectos pueden estar asociados a un único repositorio de código. Un despliegue es el resultado de una compilación exitosa del proyecto. Cada despliegue se genera automáticamente al realizar acciones como subir un commit a un repositorio conectado o utilizar la interfaz de línea de comandos (CLI) de Vercel. Cada instancia de despliegue recibe una URL única.

El proyecto es la aplicación base, mientras que el despliegue es una versión específica y funcional de dicha aplicación, lista para ser accedida en línea. Para iniciar, el proceso consiste en crear un proyecto, ya sea importando uno existente o utilizando una de las plantillas disponibles.

Proceso de Despliegue con Plantillas en Vercel

  1. Selección de una Plantilla: Explorar el catálogo de plantillas en la plataforma de Vercel.
  2. Despliegue de la Plantilla: Inicie el proceso haciendo clic en el botón correspondiente.
  3. Conexión con el Proveedor de Git: Vercel crea un nuevo repositorio conectado.
  4. Ejecución del Despliegue: Vercel inicia automáticamente la compilación.
  5. Visualización del Proyecto: Se asigna un dominio al despliegue.
  6. Clonación Local del Proyecto: Clone el repositorio en su máquina.

Importar un Proyecto Existente a Vercel

  1. Conectar con el Proveedor de Git: Seleccione GitHub, GitLab o BitBucket.
  2. Importar el Repositorio: Seleccione su proyecto de la lista.
  3. Configurar Opciones: Personalice comandos o variables si es necesario.
  4. Desplegar el Proyecto: Presione "Deploy".
  5. Ver el Despliegue: Acceda al dominio generado.

Dominios en Vercel

Puede comprar un dominio nuevo directamente en Vercel o usar un dominio existente.

  • Dominio apex (ej. ejemplo.com)
  • Subdominio (ej. blog.ejemplo.com)
  • Dominio wildcard (ej. *.ejemplo.com)

Cómo Usar un Dominio Existente en Vercel

  1. Añadir el Dominio: En la configuración del proyecto.
  2. Configurar los Registros DNS: Según el tipo de dominio (A, CNAME, TXT o Nameservers).

Colaboración en Vercel

Vercel despliega automáticamente cada commit que se sube. El flujo de trabajo colaborativo incluye:

  • Producción: Rama principal.
  • Vista previa: Para cualquier otra rama.

Crear un Despliegue de Vista Previa

  1. Haz tus cambios en una nueva rama.
  2. Sube tus cambios y crea un pull request.
  3. Vercel genera un enlace de vista previa automáticamente.

Comentar y Compartir Vistas Previas

  • Abrir y autenticar: Activa la barra de comentarios.
  • Compartir con colaboradores: Enviar enlace de vista previa.
  • Comentar: Los colaboradores pueden dejar comentarios en la vista previa.
